前端书籍书单

原文链接：http://www.vergilius.me/html-css-bestpractice
作者：vergilius 日期：2017-02-18

根据在网上搜到的书单，我将以前阅读过的前端书籍整理了一下：

Pro CSS and HTML Design Patterns

  • https://book.douban.com/subject/2064791/

内容很充实，就是稍显啰嗦，同一个例子，左边对齐的说了一遍，一模一样的右边对齐又再说一遍。

Object-Oriented JavaScript

  • https://book.douban.com/subject/3182419/

Professional JavaScript for Web Developers

  • https://book.douban.com/subject/3346905/
  • 一般讲解某种语言的书在解释代码部分分为三个级别：
    1 把大段大段无用的代码（比如html和css）copy到书中，占据大部分篇幅，每次讲到新的知识点又把前面的代码重新copy一遍，真正的干货却少得可怜。
    2 适当的写一些代码，但还是避免不了重复。
    3 所有的代码都很紧凑精简，新增的代码是为了解释某个知识点才写出来，其他相关的代码都用 伪代码来代替，没有半点废话。
    这本书从头到尾几乎都是干货。讲解某个知识点的时候 或者 解释一种模式或者方法的时候总会把与之相关的所有可能性和路径全部都列举出来，然后逐一讲解每种方法的优劣和前世今生，对比然后按照步骤一步一步的由潜入深，最后得出最优解决方案。这样就能理解代码为什么要这样写，这样写有什么好处，那样写有什么坏处，哪种写法的效率最高，哪种写法会有漏洞，可见作者js的功底非常扎实。
    另外本书还指出了无数ie的bug。第 4章(Execution Context and Scope) 第6和第7章的图， 18章(Advanced timers和custom events)，第 20章（Performance）是整本书的精华。

JavaScript权威指南

  • https://book.douban.com/subject/2228378/

JavaScript : The Good Parts

  • https://book.douban.com/subject/3332698/

很精简，将JavaScript语言里好的部分和不好的部分都讲得非常清楚，写代码时候可以放在手头做为参考。

编写可维护的JavaScript

  • https://book.douban.com/subject/21792530/

这本书前半部分的内容和《JavaScript : The Good Parts》有点重复，都是讲如何正确使用js的特性，而且最好是先读一下《JavaScript : The Good Parts》再读这本书，因为书中有些地方直接引用和提到了前者，可能因为两人都是雅虎的同事吧。后半部分可以很好的帮助自己从无到有建立起自己的js build环境。

高性能网站建设指南

  • https://book.douban.com/subject/3132277/

非常精简的一本书，对组件并行下载，下载顺序，文件合并等内容都给出了如何优化的建议，整本书都是干货。

高性能网站建设进阶指南

  • https://book.douban.com/subject/4719162/

第一章稍微鸡肋一点，可有可无。

Principles of Object-Oriented JavaScript

  • https://book.douban.com/subject/25831404/

Zakas 的书，相对于他的《Professional.JavaScript.for.Web.Developers》这本书更多的介绍了js的最新的特性。
